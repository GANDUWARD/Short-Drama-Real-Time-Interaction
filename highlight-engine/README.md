# 短剧高光检测与内容理解

这个项目面向短剧内容分析，围绕“剧情高光检测”构建一套可训练、可推理、可评测的多模态能力底座。系统结合算法与视频大模型能力，对短剧视频进行内容理解，识别不同类型的剧情高光点，并输出结构化结果，作为后续即时互动、内容推荐、切条分发、剧情摘要和运营分析的基础数据来源。

## 目标定位

- 面向短剧场景，识别剧情中的关键高光片段
- 对高光内容进行时间定位、强度评分和文本描述
- 结合模型推理与结构化输出，为后续互动玩法提供基础数据
- 支持继续扩展到剧情理解、片段检索、自动摘要、角色关系分析等任务

## 核心能力

- **高光检测**：识别视频中的关键剧情片段，输出起止时间
- **高光评分**：对高光强度进行打分，支持后续排序、筛选和触发策略
- **内容理解**：为高光片段生成简洁文本描述，帮助理解事件内容
- **多类型剧情高光识别**：可面向冲突爆发、情绪反转、关键台词、关系推进、悬念揭示、反转节点等高光类型进行建模
- **统一训练与评测**：支持高光检测、时序定位、视频描述等相关任务的训练、推理与评测

## 输出结果

当前推理脚本已经支持输出结构化高光结果，典型字段包括：

- `timestamps`：高光片段时间戳
- `scores`：高光显著性分数
- `captions`：高光内容描述
- `highlights`：按片段聚合后的结构化结果

这些结果可以直接作为后续业务层的基础数据，例如：

- 即时互动触发点生成
- 短剧切条与精彩片段提取
- 高光剧情卡片生成
- 用户兴趣点分析
- 互动问答或剧情分支推荐

## 技术思路

项目以多模态视频理解模型为基础，将视频建模为一系列时序事件，并联合建模：

- 视频帧视觉信息
- 时间位置编码
- 高光强度编码
- 文本语义描述

这样可以把“看到了什么”“发生在什么时候”“是不是高光”“高光内容是什么”统一到一套生成式框架里，适合短剧内容理解与高光挖掘场景。

## 项目结构

- `scripts/inference/`：推理脚本
- `scripts/train/`：训练与微调脚本
- `model/`：多模态模型相关实现
- `eval/`：通用评测入口
- `metrics/`：各类任务指标实现
- `prompts/`：推理与评测提示词

```bash
bash install_requirements-npu.sh
```

## 推理

高光检测的单视频推理入口位于 `scripts/inference/` 目录。

示例：

```bash
python scripts/inference/<highlight_inference_script>.py \
  --model-path /path/to/model \
  --video-path /path/to/video.mp4 \
  --query "找出这段短剧里最关键的剧情高光片段" \
  --output-json /path/to/output.json \
  --device cuda
```

输出的 `json` 中会包含高光时间戳、分数和文本描述，适合作为后续业务处理输入。

## 训练

项目保留了完整的预训练、阶段训练和下游微调脚本。运行前需要先根据本地环境修改脚本中的模型路径、数据路径和输出路径。

### Stage 1

```bash
bash scripts/train/pretrain-128.sh
```

### Stage 2

```bash
bash scripts/train/sft-128.sh
```

### 下游微调

```bash
bash scripts/train/sft-youcook2.sh
```

如果后续要专门面向短剧场景建设能力，建议优先补充：

- 短剧高光标注数据
- 高光类型标签体系
- 角色/关系/冲突事件标注
- 面向互动触发的业务标签

## 评测

指标实现目录：

- 时序视频定位指标
